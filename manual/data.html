WD Attributes Tools

The WD Attributes Tools consists of a JavaScript library that automates some methods of the WD JavaScript Tools HTML object as attributes to be inserted into HTML elements.

The tool gives HTML more power by integrating javascript methods into the code itself, reducing the demand for scripts.

Attribute

The tool uses the HTML attribute data to define which action will be implemented. The attribute structure has the prefix data-wd- followed by the tool name as follows:

data-wd-toolName=value

The toolName represents the name of the tool to be implemented in the element having the following possibilities:

  • action
  • click
  • data
  • device
  • filter
  • load
  • mask
  • page
  • repeat
  • request
  • sort
  • sort-col

Each attribute is sensitive to a specific event whose possibilities depend on the attribute. The target element of the engine also depends on the attribute. See menu for details.

Value

The value of the attributes depends on the tool used and can be in the following ways:

Primitive value

These are simple textual values ​​that can represent a numeral, a name, or code:

data-wd-tool=1

Action and argument

The attribute may require the indication of a subaction that may or may not require an argument. The subaction will be represented by its name followed by the characters {}. If this subaction requires argument, this value must be entered between the characters {}. The subaction can be single or multiple:

data-wd-tool=subaction{argument}
data-wd-tool=subaction1{argument1}subaction2{argument2}

Actions and targets

The attribute may require that a target be set for the engine. The target will be represented by CSS selector and will be defined in the subargument argument $.

data-wd-tool=${div, .class1, #id2}
data-wd-tool=subaction{argument}${div, .class1, #id2}
data-wd-tool=subaction1{argument1}subaction2{argument2}${div, .class1, #id2}

Groupings

The attribute may allow different subactions to be applied to different targets. For this situation, you will need to group the subactions with their targets. The separation of the groups will be made by the character &:

data-wd-tool=subaction1{argument1}${div, .class1, #id1}&subaction2{argument2}${.class2, #id1}
