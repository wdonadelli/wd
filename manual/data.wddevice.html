The purpose of the attribute is to make the element containing it responsive by applying certain class values ​​to deoender of the screen size of the device:

data-wd-device=desktop: className&mobile: className&tablet: className&phone: className

Each screen size can receive a list of classes. The class list must be separated by whitespace while each screen size must be separated by the & character. It is not mandatory to enter all screen sizes in the attribute. Four screen size options are supported:

desktop:
For screen sizes starting at 768px.
In this case, all classes listed in the options below will be removed from the element's class attribute while the classes listed in this option will be added to the element's class attributes.
mobile:
For screen sizes smaller than 768px.
In this case, all classes listed in the above option will be removed from the element's class attribute while the classes listed in this option will be added to the class attribute of the element.
tablet:
For screen sizes from 600 to 767px
In this case, all classes listed in the first and last options will be removed from the element's class attribute while the classes listed in this option will be added to the class attribute of the element.
phone:
For screen sizes smaller than 600px.
In this case, all classes listed in the first and second-to-last options will be removed from the element's class attribute while the classes listed in this option will be added to the class attribute of the element.

See the method HTML.class to understand the source of its operation!

Beware of conflicts!

Target:

  • Element content.

Sensitivity:

  • When loading the document.
  • When defining the attribute on the element (See the HTML.data method).
  • After external data loading procedures (repeat/load).
  • When to change the screen size.

Note:

The data-wd-desktop, data-wd-tablet and data-wd-phone attributes are obsolete.
