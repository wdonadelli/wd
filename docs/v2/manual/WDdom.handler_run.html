WDdom.handler

It is a method that sets triggers for events to elements defined in the input. It is a standalone method that uses the corresponding attribute of the element to assign the methods to execute when the event is fired.

wd(dom).handler(list)

The list argument is an object whose keys and values ​​respectively represent the names of events and their methods. Event names are case insensitive and may or may not have the prefix on:

list = {onclick: method1}; /*or*/ list = {Click: method1};

If there is more than one method to be set on the trigger, then the methods must be entered in arrays. Setting the method value to null will remove all triggers in the event:

list = {onclick: [method1, method2, method3], mouseOver: null};

If you want to exclude a specific method, you must enter the prefix - to the event name:

list = {-CLICK: [method1, method2]};

If the list argument is null, all triggers defined by the method will be removed from the element from all events.

WDdom.run


		
It is a method that executes the function entered in the argument in the same amount of existing elements, that is, if it is a list of nodes with n elements, the function will be performed n times.


		
wd(dom).run(method)

		
		
The method argument is the function to be executed. The reported method accepts an argument representing the nth element.
